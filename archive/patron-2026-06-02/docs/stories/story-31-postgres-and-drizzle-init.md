# Story 31 — Neon Postgres connection + Drizzle config + first migration

**Epic:** Epic 2 — Backend Foundation
**Estimated:** ~1.5h
**Depends on:** story-30-hono-skeleton

## BDD Acceptance Criteria

```
Given env var POSTGRES_URL is set to a valid Neon connection string
When `pnpm --filter @patron/api drizzle:generate` runs
Then a SQL migration file is created under apps/api/src/db/migrations/
And it includes `CREATE TABLE meta` (the first migration that establishes a schema_version row)
And exit code is 0

Given a fresh Neon branch DB and POSTGRES_URL pointing at it
When `pnpm --filter @patron/api drizzle:migrate` runs
Then the migration applies cleanly (exit code 0)
And `psql $POSTGRES_URL -c "SELECT version FROM meta;"` returns a single row

Given the api server boots after migrations
When the connection pool initialises
Then `SELECT 1` succeeds within 2 seconds
And a Pino log line is emitted: {"level":"info","msg":"db connected","ssl":true}
And /health endpoint now reports `{"ok":true,"db":"ok",...}`

Given a Vitest test imports `db` from `apps/api/src/db/client.ts`
When the test runs against a Neon test branch (env var DATABASE_URL_TEST)
Then queries execute against the test branch (not prod)
And `pnpm --filter @patron/api test` exits 0
```

## File modification map

- `apps/api/src/db/client.ts` — NEW — exports `db` (Drizzle instance over `@neondatabase/serverless` driver); reads `POSTGRES_URL` from env; uses the WebSocket driver in production + Node http driver in tests; pings `SELECT 1` on boot
- `apps/api/src/db/schema/index.ts` — NEW — barrel export for all schema files (empty for now; populated by stories 32-33)
- `apps/api/src/db/schema/meta.ts` — NEW — Drizzle schema for a `meta` table tracking `{schema_version int, applied_at timestamptz default now()}`; provides a single-row "is the DB initialised?" check
- `apps/api/drizzle.config.ts` — NEW — Drizzle Kit config: `dialect: 'postgresql'`, `schema: './src/db/schema/index.ts'`, `out: './src/db/migrations'`, `dbCredentials: { url: process.env.POSTGRES_URL! }`
- `apps/api/src/db/migrations/0000_initial.sql` — NEW (generated by drizzle-kit) — first migration: creates `meta` table + inserts initial row
- `apps/api/src/db/migrate.ts` — NEW — programmatic migrator (used by `drizzle:migrate` script + by CI test setup); uses `migrate()` from `drizzle-orm/neon-http/migrator`
- `apps/api/src/routes/health.ts` — UPDATE — add a `db: 'ok' | 'down'` field by running a quick `SELECT 1` with 500ms timeout
- `apps/api/src/lib/env.ts` — UPDATE — add `POSTGRES_URL: z.string().url()`, `DATABASE_URL_TEST: z.string().url().optional()`
- `apps/api/package.json` — UPDATE — deps: `drizzle-orm`, `drizzle-kit` (devDep), `@neondatabase/serverless`, `ws`; scripts: `drizzle:generate` (`drizzle-kit generate`), `drizzle:migrate` (`tsx src/db/migrate.ts`), `drizzle:studio` (`drizzle-kit studio`)
- `apps/api/vitest.config.ts` — NEW — Vitest config; `setupFiles: ['./src/__tests__/setup.ts']`
- `apps/api/src/__tests__/setup.ts` — NEW — Vitest global setup; if `DATABASE_URL_TEST` is set, switches POSTGRES_URL pointer for the test run; runs `migrate()` before suite
- `apps/api/src/__tests__/db.test.ts` — NEW — round-trip Vitest test: insert into `meta`, select it back

## Shell verification

```bash
cd apps/api

# Drizzle generate produces a migration file
pnpm drizzle:generate
ls src/db/migrations/*.sql | head -1 | xargs test -f

# Migration applies cleanly
pnpm drizzle:migrate
test $? -eq 0

# DB connection works
psql "$POSTGRES_URL" -c "SELECT version FROM meta;" | grep -E '^\s+[0-9]+'

# Health endpoint reports db ok
pnpm dev &
DEV_PID=$!
sleep 3
curl -sf http://localhost:3001/health | jq -e '.db == "ok"'
kill $DEV_PID
wait $DEV_PID 2>/dev/null || true

# Vitest passes against test branch
DATABASE_URL_TEST="${DATABASE_URL_TEST:-$POSTGRES_URL}" pnpm test
test $? -eq 0
```

## Notes

- Per **ADR-009** (architecture.md): **Neon** is the chosen Postgres provider. Use the `@neondatabase/serverless` driver — NOT `pg`. The serverless driver supports both WebSocket (long-lived connections) and HTTP (per-query, edge-friendly) modes; default to WebSocket for the long-running Hono server.
- Per **ADR-009**: every PR gets a Neon branch DB for tests. The `DATABASE_URL_TEST` env is set by CI to a branch URL; locally, devs can either use the same prod URL (safe — tests use `BEGIN; ROLLBACK;` boundaries) or spin up a personal branch via the Neon dashboard.
- Drizzle 2026 API: dialect is `'postgresql'` (not `'pg'`); use the `drizzle-orm/neon-http` or `drizzle-orm/neon-serverless` adapters depending on connection mode.
- Per architecture.md "Context7 library research rule": coding agent MUST query Context7 for Drizzle + Neon docs before writing the client. Drizzle's API has shifted across minor versions.
- The `meta` table is a sentinel — its presence + a row means "schema has been initialised". Subsequent migrations (story-32, story-33) update the `version` value.
- File MUST stay under 400 LOC each.
- Test discipline: every DB test wraps in a transaction that rolls back on completion (Vitest's `beforeEach`/`afterEach`); never leak rows between tests.
- `pnpm drizzle:studio` (optional) gives a local web UI for schema browsing — useful during stories 32-33.
