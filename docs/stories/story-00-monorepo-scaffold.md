## Acceptance criteria (BDD)

```
Given a clean clone of the repo
When `pnpm --version` runs
Then exit code is 0 and version is ≥ 10

Given Node is installed
When `node --version` runs
Then exit code is 0 and version is ≥ 22

Given the repo root has package.json + pnpm-workspace.yaml
When `pnpm install` runs
Then pnpm-lock.yaml is generated and exit code is 0

Given the workspace is configured
When `pnpm list -r --depth -1` runs at root
Then it returns exit code 0 (no error parsing workspace globs) and includes the root project

Given .gitignore is committed
When `git status` runs after a `pnpm install`
Then node_modules is NOT in the staged diff

Given pnpm-workspace.yaml is committed
When `node -e "const fs = require('fs'); const y = fs.readFileSync('pnpm-workspace.yaml','utf8'); for (const g of ['apps/*','packages/*','packages/providers/*']) if (!y.includes(g)) process.exit(1)"` runs
Then exit code is 0 (all three globs present)
```

## Shell verification

```bash
test -f package.json
test -f pnpm-workspace.yaml
test -f .npmrc
test -f .gitignore
test -f LICENSE
test -d apps
test -d packages
test -d packages/providers
test -d contracts

# Tooling versions
pnpm --version       # ≥ 10
node --version       # ≥ 22

# Workspace install completes
pnpm install
test $? -eq 0
test -f pnpm-lock.yaml

# .gitignore prevents node_modules from staging
git status --porcelain | grep -q "node_modules" && { echo "FAIL: node_modules staged"; exit 1; } || echo "OK"

# pnpm-workspace.yaml has all three globs
node -e "
  const fs = require('fs');
  const y = fs.readFileSync('pnpm-workspace.yaml', 'utf8');
  for (const g of ['apps/*', 'packages/*', 'packages/providers/*']) {
    if (!y.includes(g)) process.exit(1);
  }
"

# pnpm resolves the workspace cleanly
pnpm list -r --depth -1
test $? -eq 0
```

## Notes for coding agent

- Use pnpm (not bun, not npm) per ADR-001 (amended 2026-06-09). `pnpm init` is the bootstrap; do NOT use `npm init` or `bun init`.
- pnpm reads workspace globs from `pnpm-workspace.yaml`, NOT from `package.json#workspaces`. Putting them in `package.json` is a silent no-op for pnpm — don't.
- `.npmrc` settings (`strict-peer-dependencies=true`, `auto-install-peers=false`) match ADR-018's peer-dep strategy: every adapter package declares framework SDKs as `peerDependencies`, and pnpm should *refuse* to install if the consumer hasn't installed them. `auto-install-peers=false` keeps the lockfile honest.
- `packageManager` field in `package.json` (e.g. `"pnpm@9.x.x"`) lets Corepack auto-provision the pinned pnpm — anyone with Corepack-enabled Node gets the right version automatically.
- License is MIT — copy from https://opensource.org/license/mit verbatim.
- README is a placeholder for now (`Concierge — autonomous AI agent for Mantle. Setup instructions in story-200-readme-finalize.`).
- The `contracts/` folder gets Foundry-initialized in story-03 — leave it as `.gitkeep` for now.
- `apps/` will hold `web/` + `mcp/` + `worker/`; `packages/` will hold `sdk/`, `shared/`, `ui/`, `skill/`; `packages/providers/` will hold the 7 `@mpilot/<provider>` packages.
