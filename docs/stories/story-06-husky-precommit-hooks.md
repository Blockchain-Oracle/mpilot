## File modification map

- `package.json` — UPDATE — add `lint-staged` + `@commitlint/cli` + `@commitlint/config-conventional` as `devDependencies` (pinned EXACT, no `^` or `~`). `husky` was already added in story-01 with `prepare: "husky"` (v9 idiom — `husky install` is the v8 form; spec patched 2026-06-09).
- `.husky/pre-commit` — UPDATE (story-01 created the original) — runs `pnpm exec lint-staged` (staged-file Biome) + `node scripts/check-file-loc.mjs` (full-tree LOC guard) + `pnpm run typecheck` (workspace-wide) + `gitleaks detect ...` IF `gitleaks` is on PATH (conditional — matches the slither smoke-test pattern; CI's story-07 security workflow runs gitleaks unconditionally). Hook stays executable. Spec patched 2026-06-09: `pnpm scripts/X.mjs` doesn't execute .mjs files — use `node scripts/X.mjs`.
- `.husky/commit-msg` — NEW — runs `pnpm exec commitlint --edit $1` — enforces Conventional Commits.
- `.husky/pre-push` — NEW — runs LOC smoke test (`node scripts/test-check-file-loc.mjs`) + a quick `forge build` (if `contracts/` files differ vs origin). Catches script regressions before push.
- `.lintstagedrc` — NEW — JSON: `{"**/*.{ts,tsx,js,jsx,mjs,json,md,yml,yaml}": ["pnpm exec biome check --write --no-errors-on-unmatched"]}`. Spec patched 2026-06-09: Biome 2.x uses `--write` not `--apply`; the single `biome check` invocation covers lint + format + import sort (no need for a separate format pass).
- `commitlint.config.mjs` — NEW — extends `@commitlint/config-conventional`. ERROR-level: `type-enum` locked to `[feat, fix, docs, style, refactor, perf, test, build, ci, chore, revert]`, `subject-case: [2, 'never', ['upper-case', 'sentence-case']]` (no Title Case in subjects). WARN-level (`[1, 'always', [...]]`): `scope-enum` covers the canonical package set per `docs/architecture.md` § Repo structure — `[sdk, shared, agent, tools, ui, skill, react, react-ui, react-assistant-ui, react-copilotkit, vercel-ai, openai, langchain, agentkit, mcp, providers, aave, dex, ethena, ondo, meth-staking, lifi, erc8004, web, worker, contracts, docs, ci, deps]`. Warn (not error) so a sensible new scope doesn't block a PR — typos still surface in the warning output. Spec patched 2026-06-09 to expand from the original 12-scope list (which omitted real packages like `agent`, `tools`, and individual provider names).
- `.gitleaks.toml` — NEW — Gitleaks config for additional regex patterns specific to mPilot (Anthropic `sk-ant-`, Pimlico `pim_`, ZeroDev project IDs, Pinata JWTs, Privy keys, Sepolia private keys with `0x...64-hex` shape inside `.env*` paths).
- `.husky/_/.gitignore` — generated by `husky install`.

## Acceptance criteria (BDD)

```
Given husky is installed via `pnpm install`
When `ls .husky/` runs
Then it lists `pre-commit`, `commit-msg`, `pre-push` files (all executable)

Given a TS file has Biome violations and is staged
When `git commit -m "feat(sdk): test"` runs
Then the commit is BLOCKED by `pnpm exec lint-staged` and Biome reports the violations

Given a non-conventional commit message ("WIP fix things") is attempted
When the commit hook fires
Then commit-msg rejects with exit code 1 and a message naming the missing type

Given a valid conventional commit ("feat(sdk): add mPilot skeleton") is staged
When `git commit` runs
Then exit code is 0 and the commit is created

Given a file >400 LOC is staged
When the commit hook fires
Then `scripts/check-file-loc.mjs` BLOCKS the commit (defense-in-depth with story-01's Biome rule)

Given commitlint config is valid
When `node -e "import('./commitlint.config.mjs').then(c => console.log(c.default.rules['type-enum'][2]))"` runs
Then output is an array containing "feat", "fix", "docs", "style", "refactor", "perf", "test", "build", "ci", "chore", "revert"

Given commitlint scope-enum is set
When the same node command checks scope-enum
Then output contains "sdk", "providers", "contracts", "ci"

Given an Anthropic API key shape (sk-ant-abc123...) is staged in a TS file
When the pre-commit hook fires
Then Gitleaks BLOCKS the commit and identifies the secret
```

## Shell verification

```bash
test -f .husky/pre-commit
test -f .husky/commit-msg
test -f .husky/pre-push
test -f .lintstagedrc
test -f commitlint.config.mjs
test -f .gitleaks.toml

# Hooks are executable
test -x .husky/pre-commit
test -x .husky/commit-msg
test -x .husky/pre-push

# Husky + lint-staged + commitlint pinned in devDependencies
node -e "
  const pkg = require('./package.json');
  for (const dep of ['husky', 'lint-staged', '@commitlint/cli', '@commitlint/config-conventional']) {
    const v = pkg.devDependencies?.[dep];
    if (!v) { console.error('Missing dep:', dep); process.exit(1); }
    if (v.startsWith('^') || v.startsWith('~')) { console.error('Unpinned:', dep, v); process.exit(1); }
  }
"

# Commitlint config exports valid rules
node --input-type=module -e "
  import('./commitlint.config.mjs').then(({ default: cfg }) => {
    const types = cfg.rules['type-enum']?.[2] ?? [];
    if (!Array.isArray(types) || types.length < 10) process.exit(1);
    const scopes = cfg.rules['scope-enum']?.[2] ?? [];
    if (!scopes.includes('sdk') || !scopes.includes('providers') || !scopes.includes('contracts')) process.exit(1);
  }).catch(() => process.exit(1));
"

# Pre-commit hook references the LOC guard + Biome
grep -q "scripts/check-file-loc.mjs" .husky/pre-commit
grep -qE "(biome|lint-staged)" .husky/pre-commit

# commit-msg invokes commitlint
grep -q "commitlint" .husky/commit-msg

# pre-push runs the smoke test
grep -q "test-check-file-loc.mjs" .husky/pre-push

# Conventional-commit validator rejects bad messages
echo "WIP fix stuff" | pnpm exec commitlint
test $? -ne 0

# Conventional-commit validator accepts good messages
echo "feat(sdk): add mPilot class skeleton" | pnpm exec commitlint
test $? -eq 0

# Gitleaks runs without finding secrets in the current tree.
# Direct binary invocation (not pnpm exec — gitleaks is not on npm; install
# locally via `brew install gitleaks`). CI's story-07 security workflow
# runs gitleaks unconditionally. Local pre-commit skips with a warning
# annotation if gitleaks is missing (matches the slither smoke pattern).
if command -v gitleaks >/dev/null 2>&1; then
  gitleaks detect --no-banner --config .gitleaks.toml --redact -v --source .
  test $? -eq 0
fi
```

## Notes for coding agent

- Reference patterns: `find-evil/.pre-commit-config.yaml` (109 lines, most complete chain) + `rapid-agents/.pre-commit-config.yaml` (TS-Python hybrid). Abu's cross-project pattern enforces: format → lint → typecheck → LOC → secret-scan → commit-msg.
- **Husky is the TS-equivalent of Python's `pre-commit` framework.** Same gates, different installer. `husky install` is wired via `prepare` script so `pnpm install` auto-installs hooks for every contributor.
- **lint-staged** runs Biome only on staged files (fast). The full-tree LOC guard + typecheck run unconditionally — they're fast enough.
- **`commitlint.config.mjs`** uses ESM (mjs extension) — Node 22+ loads it correctly. Scope-enum locks scopes to known package names so PR titles + commits stay consistent with `docs/architecture.md` § Repo structure.
- **Gitleaks** is the secret-scan layer. Use Gitleaks **without license requirement** for solo dev — they're free for personal use; if mPilot grows, swap to `GITLEAKS_LICENSE` secret. Reference: `aegis/.github/workflows/security.yml:48` shows the licensed form.
- **`pre-push`** runs the smoke test (`pnpm scripts/test-check-file-loc.mjs`) — confirms `scripts/check-file-loc.mjs` still behaves correctly before any code leaves the local machine. Catches regressions that pre-commit alone misses.
- If a contributor needs to bypass in a genuine emergency: `git commit --no-verify` — but CI re-runs every hook via the equivalent steps in story-04's workflow. The escape hatch buys nothing once CI is wired. Document this in `CONTRIBUTING.md` (story-04).
- Pin all hook tool versions in `devDependencies` (no `^` or `~`) — pre-commit toolchain drift is a real source of "works on my machine" PRs (reference: `find-evil/.pre-commit-config.yaml:10-14` rationale).
